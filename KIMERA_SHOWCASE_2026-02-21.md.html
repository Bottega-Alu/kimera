Session Overview

In a single development session, Kimera was born — conceived, designed, built, branded, and published to the world. What began as a simple request for a printable status memo evolved into a complete document parsing and formatting ecosystem. Kimera fuses Markdown structure with HTML rendering, producing self-contained .md.html files that open in any browser, print perfectly on any printer, and edit seamlessly in Microsoft Word, LibreOffice, or Google Docs. No dependencies. No internet required. One file that works everywhere, forever.

Visão Geral

Kimera é um formato de documento e uma habilidade de IA que parsa e formata documentos profissionais, paginados e prontos para impressão a partir de conversas com inteligência artificial. Os arquivos de saída utilizam o formato dual .md.html — conteúdo estruturado em Markdown renderizado como HTML autônomo. Um arquivo. Zero dependências. Abra no navegador. Imprima. Edite no Word. Funciona offline, para sempre. O nome vem da Quimera, a criatura mitológica que funde seres diferentes em um só.

Badge System

OK / DONE  WARNING / P1  ERROR / P0  INFO / ACTIVE  NEW / SPECIAL

Every badge uses the triad pattern: background color + border color + text color. This ensures accessibility — readable even without color perception. Never use color alone to convey meaning.

Info Boxes

Blue / Info: General information, tips, and neutral callouts for the reader.
Green / Success: Positive outcomes, completed items, healthy status indicators.
Orange / Warning: Attention required, important caveats, P1 priority items.
Red / Error: Critical issues, P0 blockers, confidential notices, breaking changes.

Typography Scale

15pt Page Title (h1)
11pt Section Heading (h2)
9.5pt Subsection (h3)
9pt Body text — the standard reading size for content
8pt Table content, info boxes, compact sections
7pt Footer text, badge labels, metadata
6.5pt Legal text, fine print, copyright notices
